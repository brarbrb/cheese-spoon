Privacy & Transparency
Prototype status
This is a mock UI for a student project. Some buttons and data are simulated.
What we use (intended)
  • Course catalog + availability (semester offerings)
  • Prerequisites + your completed courses (from your grade sheet)
  • Public course descriptions + student review text
What we store (intended)
For the prototype, we aim to store only what’s needed to generate recommendations (e.g., course IDs / credits), and avoid storing the original uploaded file.
Transparency
Recommendations will include “why recommended” explanations (e.g., interest match, workload signals, bottleneck impact), and will make limitations visible (e.g., low review coverage).
Features
Eligibility
Filters to courses you can take this semester (prereqs + availability).
Recommendations
Ranks by interest match, course quality signals, and preference fit.
Explainability
“Why recommended” bullets + details (workload, exam/project, bottleneck impact).
